PROJECT OVERVIEW
================================================================================
FlipTrack is a fully functional Next.js 14 CRM and profit tracking application
specifically designed for French flea market sellers ("brocanteurs/flippers").

The application allows users to:
- Track items from purchase to sale
- Calculate profit and ROI automatically
- Manage expenses and costs
- Upload and organize photos
- Monitor inventory and sales performance

================================================================================
TECHNOLOGY STACK
================================================================================
Frontend:     Next.js 14 with App Router
Styling:      Tailwind CSS with custom color palette
Database:     Supabase (PostgreSQL)
Auth:         Supabase Authentication
Storage:      Supabase Storage (for photos)
Deployment:   Vercel-ready

================================================================================
DATABASE SCHEMA
================================================================================

profiles (user profiles)
  - id (UUID) - Primary key
  - email (TEXT) - Unique email
  - display_name (TEXT)
  - plan (TEXT) - 'free' or 'pro'
  - stripe_customer_id (TEXT) - For future payments
  - created_at, updated_at (TIMESTAMP)

items (articles/flips)
  - id (UUID) - Primary key
  - user_id (UUID) - Foreign key to profiles
  - title, description (TEXT)
  - category (TEXT) - meuble, decoration, vintage, etc.
  - condition (TEXT) - a_restaurer, bon_etat, comme_neuf
  - status (TEXT) - stock, restauration, liste, vendu, expedie
  - purchase_price, purchase_date, purchase_source
  - restoration_cost (DECIMAL)
  - sale_price, sale_date, sale_platform
  - platform_fees, shipping_cost (DECIMAL)
  - net_profit, roi_percent (DECIMAL)
  - photos (TEXT[]) - Array of photo URLs
  - notes (TEXT)
  - created_at, updated_at (TIMESTAMP)

expenses (expense tracking)
  - id (UUID) - Primary key
  - user_id (UUID) - Foreign key to profiles
  - amount (DECIMAL)
  - category (TEXT) - carburant, materiaux, emplacement, etc.
  - description (TEXT)
  - date (DATE)
  - item_id (UUID) - Optional link to items
  - created_at (TIMESTAMP)

SUPABASE SETUP
================================================================================

Required Configuration:

1. Tables: Create using database-setup.sql
   - profiles
   - items
   - expenses

2. Row-Level Security: Enabled for all tables
   - Users can only access their own data

3. Storage Bucket: "item-photos"
   - Make it public
   - Add storage RLS policies (see database-setup.sql)

4. Auth: Supabase Auth enabled
   - Email/password method
   - Email confirmation optional

Environment Variables:
  NEXT_PUBLIC_SUPABASE_URL
  NEXT_PUBLIC_SUPABASE_ANON_KEY

================================================================================
CALCULATION FORMULAS
================================================================================

Net Profit:
  net_profit = sale_price - (purchase_price + restoration_cost + platform_fees + shipping_cost)

ROI Percentage:
  roi_percent = (net_profit / (purchase_price + restoration_cost)) × 100

Platform Fees Examples:
  - LeBonCoin: sale_price × 0.0499
  - Vinted: 0
  - Selency: sale_price × 0.15
  - Facebook: 0

================================================================================
GETTING STARTED
================================================================================

1. Installation:
   npm install

2. Environment Setup:
   cp .env.local.example .env.local
   # Add your Supabase credentials

3. Database Setup:
   - Copy database-setup.sql
   - Paste into Supabase SQL Editor
   - Create item-photos bucket in Storage

4. Run Development Server:
   npm run dev

5. Build for Production:
   npm run build

6. Deploy:
   Deploy to Vercel with environment variables
